## 1. 개요 (Overview)

### 1.1 제품명
**SayKnow**

- 어원: `say`(말하다) + `know`(알다) — 말하면 바로 이해된다는 컨셉.
- 메뉴 바/트레이 상주형 유틸리티 특성상, 이름에 "Translate"를 붙이지 않아도 기능 전달이 충분하다고 판단.
- 표기: 마케팅/문서에서는 `SayKnow`, 코드/패키지명 등 식별자에서는 `sayknow` 사용.

### 1.2 한 줄 설명
사용자가 타이핑하는 즉시 AI가 자동으로 번역해주는 macOS / Windows 데스크톱 유틸리티 앱.

### 1.3 배경 및 문제 정의
- 외국어로 메일/메시지/문서를 작성할 때, 매번 번역기 사이트(Papago, Google Translate 등)에 들어가 복사·붙여넣기를 반복해야 하는 번거로움이 있다.
- 기존 번역기는 단순 사전형 번역에 가까워, 문맥을 반영한 자연스러운 번역이 부족하다.
- 데스크톱에서 항상 떠 있으면서, 타이핑 흐름을 끊지 않고 즉시 번역 결과를 보여주는 가벼운 도구가 필요하다.

### 1.4 목적 (Goal)
"타이핑 → 번역 결과 확인 → 다듬기"의 흐름을 한 화면에서 끊김 없이 처리할 수 있게 한다.

## 2. 타깃 사용자 (Target Users)

| 페르소나 | 설명 | 주요 사용 시나리오 |
|---------|------|------------------|
| 직장인 (영어 이메일) | 한국어로 쓰고 영어로 보내야 하는 사용자 | 한국어 초안 → 자연스러운 영어 번역 → 톤 조정 |
| 학생 / 연구자 | 영어 논문/자료를 자주 다루는 사용자 | 영어 → 한국어 빠른 이해, 한국어 → 영어 작문 |
| 콘텐츠 크리에이터 | 다국어 SNS/블로그 운영자 | 다국어 동시 작성, 톤 매칭 |

## 3. 핵심 기능 (Core Features)

### 3.1 메뉴 바 / 시스템 트레이 실행
- **macOS**: 화면 우측 상단 **메뉴 바(Menu Bar)** 영역에 아이콘 상주.
  - 클릭 시 드롭다운 형태의 작은 번역 창이 펼쳐짐.
  - 시스템 시작 시 자동 실행 옵션 제공.
- **Windows**: 작업 표시줄 우측 **시스템 트레이(System Tray)** 에 아이콘 상주.
  - 좌클릭 시 번역 창 토글, 우클릭 시 설정 메뉴.
- 아이콘에서 글로벌 단축키(예: `⌘ + Shift + T` / `Ctrl + Shift + T`)로 창 열기/닫기.

### 3.2 자동 번역 (Auto Translate) — 핵심 기능 ①
- 입력창에 타이핑하면 **타이핑이 멈춘 시점부터 일정 시간(debounce, 예: 500ms~800ms)** 후 자동으로 번역 호출.
- 번역 결과는 입력창 아래(또는 옆) 결과창에 실시간으로 갱신.
- 입력 도중에도 결과창의 이전 번역은 유지(깜빡임 방지).
- 너무 짧은 입력(예: 1~2글자)은 호출 생략하여 API 비용/요청 절감.
- 원본 언어는 자동 감지(예: Claude/GPT가 감지) + 사용자가 대상 언어 지정.

### 3.3 수정 번역 (Refine Translate) — 핵심 기능 ②
- 자동 번역 결과 위에서 사용자가 추가 지시를 내려 톤·스타일·정확도를 다듬을 수 있는 기능.
- 예시 프리셋:
  - "더 정중하게 (More polite)"
  - "더 캐주얼하게 (More casual)"
  - "비즈니스 메일 톤"
  - "더 짧게"
  - "직역에 가깝게"
- 자유 입력 프롬프트도 지원 ("이 문장을 좀 더 다정하게 바꿔줘" 등).
- 수정 결과는 새 결과로 대체하거나, 히스토리로 이전 버전 비교 가능.

### 3.4 외부 AI API 연동 (BYOK + 커스텀 엔드포인트) — 핵심 기능 ③
사용자가 자신의 API 키를 직접 입력해 다양한 AI 모델을 사용할 수 있도록 한다.
OpenRouter처럼 여러 모델을 한 키로 라우팅하는 서비스도 지원해야 한다.

**3.4.1 지원 제공자 (Built-in Presets)**
- Anthropic (Claude)
- OpenAI (GPT-4o, GPT-4o-mini 등)
- **OpenRouter** — 한 키로 수십 종의 모델(Claude, GPT, Gemini, Llama 등) 사용 가능. 우선 지원.
- Google (Gemini)
- DeepL
- Groq, Together AI, Fireworks 등 OpenAI 호환 API

**3.4.2 커스텀 엔드포인트 (Custom Endpoint)**
사용자가 직접 다음 정보를 입력해 임의의 OpenAI 호환 엔드포인트를 등록할 수 있어야 한다.
- Provider 이름 (사용자 지정 라벨)
- Base URL (예: `https://openrouter.ai/api/v1`, `http://localhost:11434/v1` 등)
- API Key
- 모델명 (예: `anthropic/claude-3.5-sonnet`, `openai/gpt-4o-mini`, `meta-llama/llama-3.1-70b`)
- (선택) 추가 헤더 — OpenRouter의 `HTTP-Referer`, `X-Title` 등 커스텀 헤더 입력 가능

이 구조는 로컬 LLM(Ollama, LM Studio 등 OpenAI 호환 서버)도 동일한 흐름으로 사용 가능하게 한다.

**3.4.3 모델 프로필 (Model Profiles)**
- 여러 모델을 프로필로 저장하고 빠르게 전환할 수 있는 UI 제공.
- 예시 프로필:
  - "빠름" → OpenRouter / `openai/gpt-4o-mini`
  - "고품질" → OpenRouter / `anthropic/claude-3.5-sonnet`
  - "로컬" → Ollama / `qwen2.5:7b`
- 자동 번역과 수정 번역에 서로 다른 프로필을 지정할 수 있도록 한다 (예: 자동은 빠른 모델, 수정은 고품질 모델).

**3.4.4 API 추상화 계층**
- 내부적으로 모든 제공자를 공통 인터페이스로 호출 (예: `translate(text, from, to, options) → result`).
- 신규 제공자 추가 시 어댑터(Adapter) 한 개만 작성하면 되도록 플러그인 구조 권장.
- OpenAI 호환 API(`/chat/completions`)는 단일 어댑터로 OpenRouter, Groq, Together, Fireworks, Ollama 등을 모두 커버.

**3.4.5 API 키 보안**
- 모든 키는 OS 키체인(macOS Keychain) / Credential Manager(Windows)에 저장.
- 메모리에서도 필요한 시점에만 복호화하여 사용.
- 키 입력 UI는 마스킹 처리(`••••••••`), 표시 토글 제공.

**3.4.6 사용량/비용 가시성 (선택)**
- 응답에 포함된 token usage를 파싱해, 설정 화면에서 일/월 누적 사용량과 추정 비용을 표시.

### 3.5 부가 기능 (Should-have)
- **복사 버튼**: 결과 한 번에 클립보드 복사.
- **언어 스왑 버튼**: 원본/대상 언어 빠른 전환.
- **즐겨찾는 표현 저장**: 자주 쓰는 번역 결과 즐겨찾기.
- **히스토리**: 최근 N개의 번역 기록 보관 (로컬 저장).
- **다크 모드 / 라이트 모드** 자동 대응.

### 3.6 향후 고려 (Could-have, v2 이상)
- 시스템 전역에서 텍스트 선택 → 단축키 → 즉시 번역 (접근성 권한 필요).
- 음성 입력 → 번역.
- OCR(스크린샷 영역 번역).
- 팀 사용자별 용어집(Glossary) 동기화.

## 4. 비기능 요구사항 (Non-functional Requirements)

| 항목 | 요구사항 |
|------|---------|
| 응답 속도 | 자동 번역은 타이핑 정지 후 1.5초 이내 첫 결과 표시 목표 |
| 메모리 | 유휴 상태에서 200MB 이하 권장 |

| 개인정보 | 번역 입력 내용은 기본적으로 로컬에만 저장, 서버 보관 X |
| 접근성 | 키보드만으로 모든 핵심 기능 사용 가능 |

## 5. 기술 스택 (Tech Stack) — 후보안

> 최종 결정은 다음 단계에서 합니다. 현재는 후보만 명시.

### 5.1 프레임워크
- **추천 후보 1: Electron (TypeScript)**
  - 장점: macOS 메뉴 바 / Windows 시스템 트레이 모두 공식 API로 쉽게 구현. 자료 풍부.
  - 단점: 메모리/번들 크기 큼.
- **추천 후보 2: Tauri (Rust + Web)**
  - 장점: 가볍고 빠름. 보안 좋음.
  - 단점: 학습 곡선 있음.
- (대안) Swift(Mac) + WPF/.NET(Win) 네이티브 — 두 번 개발해야 함.

### 5.2 AI 번역 모델 (멀티 제공자 + BYOK)
- 1차 출시부터 **다중 제공자 + 사용자 자기 키(BYOK)** 구조로 설계.
- 제공자 어댑터 종류:
  - **OpenAI 호환 어댑터 1종** — OpenRouter, Groq, Together AI, Fireworks, Ollama, LM Studio 등 OpenAI `/chat/completions` 규격을 따르는 모든 엔드포인트를 처리.
  - **Anthropic 어댑터** — Claude Messages API 직접 호출.
  - **Google 어댑터** — Gemini API.
  - **DeepL 어댑터** — DeepL Translate API.
- 사용자는 설정에서 제공자별 키 + Base URL + 기본 모델을 등록하고, "모델 프로필"로 묶어 자동/수정 번역에 매핑.
- 기본 권장(추천) 프로필은 **OpenRouter** — 키 하나로 가장 다양한 모델 시도가 가능하기 때문.

### 5.3 저장소
- 로컬 설정/히스토리: SQLite 또는 JSON 파일.
- API 키: macOS Keychain / Windows Credential Manager.

## 6. UX 흐름 (User Flow)

### 6.1 첫 실행
1. 앱 설치 후 실행 → 메뉴 바/트레이에 아이콘 등장.
2. 첫 클릭 시 온보딩:
   - **AI 제공자 선택** (OpenRouter / Anthropic / OpenAI / 커스텀 ...)
   - API 키 입력 (+ 커스텀일 경우 Base URL · 모델명 입력)
   - 연결 테스트 (작은 샘플 번역 호출로 키/엔드포인트 검증)
   - 기본 언어 쌍 선택, 단축키 설정.
3. 시작 시 자동 실행 여부 선택.

### 6.1.1 제공자 추가 (설정 → AI Provider)
1. "+ 새 제공자 추가" 클릭.
2. 프리셋 선택(OpenRouter / OpenAI / Anthropic / Google / DeepL / 커스텀).
3. 프리셋 선택 시 Base URL은 자동 채움, 사용자는 API 키 + 모델명만 입력.
4. "연결 테스트" 버튼 → 성공 시 저장.
5. "모델 프로필"에서 자동 번역 / 수정 번역에 어떤 제공자·모델을 쓸지 매핑.

### 6.2 일반 사용 (자동 번역)
1. 단축키 또는 메뉴 바 아이콘 클릭 → 번역 팝업 등장.
2. 상단 입력창에 텍스트 타이핑.
3. 타이핑 정지 후 ~700ms → 하단에 자동 번역 결과.
4. 사용자는 결과를 확인하고 복사하거나 입력을 이어서 수정.

### 6.3 수정 번역
1. 자동 번역 결과 옆 "수정 (Refine)" 버튼 클릭.
2. 프리셋(정중하게/캐주얼/짧게 등) 선택 또는 자유 프롬프트 입력.
3. 새 결과가 결과창에 갱신되며, 이전 결과는 히스토리에 저장.

## 10. 리스크 및 미해결 이슈 (Open Questions)

1. **API 비용 모델**: 1차는 BYOK(사용자 자기 키)로 확정. 추후 자체 서버 프록시/구독 모델은 v2에서 검토.
2. **시스템 전역 타이핑 감지**: macOS는 접근성 권한, Windows는 글로벌 후킹 필요 → 권한 안내 UX.
3. **오프라인 모드**: 캐시된 번역만 보여줄지, 로컬 모델(예: NLLB) 옵션을 둘지.
4. **개인정보 처리방침**: 입력 내용이 외부 AI API로 전송되는 것에 대한 고지 필요.
5. **로고 / 아이콘 디자인**: 메뉴 바 친화적인 단색 모노크롬 아이콘 컨셉 필요.
